Detailed Description

This class contains all relevant information for the JNI view of CHIPDeviceController to handle all controller-related processing.

Generally it contains the DeviceController class itself, plus any related delegates/callbacks.

This function generates an operational certificate chain for the device.

The API generates the certificate in X.509 DER format.

The delegate is expected to use the certificate authority whose certificate is returned in GetRootCACertificate() API call.

The delegate will call onCompletion when the NOC certificate chain is ready.

Parameters
[in]csrElementsCSR elements as per specifications section 11.22.5.6. NOCSR Elements.
[in]attestationSignatureAttestation signature as per specifications section 11.22.7.6. CSRResponse Command.
[in]DACDevice attestation certificate received from the device being commissioned
[in]PAIProduct Attestation Intermediate certificate
[in]PAAProduct Attestation Authority certificate
[in]onCompletionCallback handler to provide generated NOC chain to the caller of GenerateNOCChain()
Returns
CHIP_ERROR CHIP_NO_ERROR on success, or corresponding error code.

Implements chip::Controller::OperationalCredentialsDelegate.

This is a synchronous Get API, where the value is returned via the output buffer.

This API can be used to retrieve a byte buffer value from the storage. There is no implied data format and and data will be stored/fetched binary. Caller is responsible to take care of any special formatting needs (e.g. byte order, null terminators, consistency checks or versioning).

Parameters
[in]keyKey to lookup
[out]bufferValue for the key
[in,out]sizeInput value buffer size, output length of value. The output length could be larger than input value. In such cases, the user should allocate the buffer large enough (>= output length), and call the API again.

Implements chip::PersistentStorageDelegate.
