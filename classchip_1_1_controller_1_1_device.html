void chip::Controller::Device::Init ( ControllerDeviceInitParams  params,
FabricIndex  fabric

Initialize the device object with secure session manager and inet layer object references.

This variant of function is typically used when the device object is created from a serialized device information. The other parameters (address, port, interface etc) are part of the serialized device, so those are not required to be initialized.

Note: The lifetime of session manager and inet layer objects must be longer than that of this device object. If these objects are freed, while the device object is still using them, it can lead to unknown behavior and crashes.

Parameters
[in]paramsWrapper object for transport manager etc.
[in]fabricLocal administrator that's initializing this device object

Initialize a new device object with secure session manager, inet layer object, and other device specific parameters.

This variant of function is typically used when a new device is paired, and the corresponding device object needs to updated with all device specifc parameters (address, port, interface etc).

This is not done as part of constructor so that the controller can have a list of uninitialized/unpaired device objects. The object is initialized only when the device is actually paired.

Parameters
[in]paramsWrapper object for transport manager etc.
[in]deviceIdNode ID of the device
[in]peerAddressThe location of the peer. MUST be of type Transport::Type::kUdp
[in]fabricLocal administrator that's initializing this device object

CHIP_ERROR chip::Controller::Device::OpenCommissioningWindow ( uint16_t  timeout,
uint32_t  iteration,
CommissioningWindowOption  option,
const ByteSpan &  salt,
Callback::Callback< OnOpenCommissioningWindow > *  callback,
SetupPayload &  setupPayload 
)

Trigger a paired device to re-enter the commissioning mode.

If an onboarding token is provided, the device will use the provided setup PIN code and the discriminator to advertise itself for commissioning availability. If the token is not provided, the device will use the manufacturer assigned setup PIN code and discriminator.

The device will exit the commissioning mode after a successful commissioning, or after the given timeout time.

Parameters
[in]timeoutThe commissioning mode should terminate after this much time.
[in]iterationThe PAKE iteration count associated with the PAKE Passcode ID and ephemeral PAKE passcode verifier to be used for this commissioning.
[in]optionThe commissioning window can be opened using the original setup code, or an onboarding token can be generated using a random setup PIN code (or with the PIN code provied in the setupPayload).
[in]saltThe PAKE Salt associated with the PAKE Passcode ID and ephemeral PAKE passcode verifier to be used for this commissioning.
[in]callbackThe function to be called on success or failure of opening of commissioning window.
[out]setupPayloadThe setup payload corresponding to the generated onboarding token.
Returns
CHIP_ERROR CHIP_NO_ERROR on success, or corresponding error
